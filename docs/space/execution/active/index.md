# Active Execution Plans

`active/` 用来存放当前仍在推进中的执行计划。

它回答的是：

- 现在仓库里有哪些计划正在推进
- 每个计划的边界和完成标准是什么
- 当前阶段是否允许直接进入实现

## 进入规则

只有在下面条件同时成立时，才把一个计划放进 `active/`：

1. 目标已经明确
2. 边界已经明确
3. 验证方式已经明确
4. 当前确实准备执行，而不是只是讨论

如果还处在探索、概念收敛或方向比较阶段，不要放进这里。

## 文件应该写什么

每个活跃计划应至少包含：

1. 目标
2. 非目标
3. 影响范围
4. 执行顺序
5. 验证方式
6. 完成标准

可以简短，但必须可执行。

## 文件不该写什么

不要在 `active/` 里写：

- 长期架构真理
- 没有时间边界的愿景
- 大量背景故事
- 与代码无关的泛泛想法

这些内容应留在根级控制文档或更上层索引，不应进入执行计划目录。

## 生命周期

- 计划进入执行前，放在 `active/`
- 计划完成并验证后，移到 `completed/`
- 如果计划失效、取消或被更小计划取代，不要长期滞留在 `active/`

## 当前阶段约束

OpenPX 当前不追求“计划很多”，而追求：

- 每个活跃计划都能被执行
- 每个活跃计划都有明确验证
- `active/` 不变成新的文档堆积区
